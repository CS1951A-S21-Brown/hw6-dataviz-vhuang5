Accesible Data Visualization:

4) In “A Comprehensive Guide to Accessible Data Visualization,” a great deal of emphasis was put on accessibility for those with visual impairments. This means picking a color palate that caters towards those with color-blindness. Those with only color blindness could easily navigate my dashboard due to my choice of colors as well as tooltips in case a specific color is easily confused with another. I also seperated bars, slices of each graph in order to allow color-blind users to better differentiate between bars/slices. However, those who are completely blind would find it difficult to use text to speach programs to navigate my dashboard. Graphs 2 and 3 have toggle elements which are not buttons, but rather custom inputs. If I were to make the dashboard easier but less aesthetically pleasing I would add buttons for every possible genre in graph 3, this way the blind user could iterate through each button for all the possible genres and have text to speach assistance reading each option to them.

5) When designing the interactive capabilities of the graphs, I have created alternative methods, such as more buttons, to guide text to speach users better than the custom inputs I have right now. However, this is a tradeoff between aesthetics and inclusiveness, which should not be a debate but if I were pushing this dashboard I would simply have two versions, one with more button options and one without. Furthermore, I could have included tooltips on the bar graph to help vision impaired users get more info about each bar, the reason I did not was simply out of time constraint.
